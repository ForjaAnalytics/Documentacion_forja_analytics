Frontend

Un SPA de React 18 sobre Vite, con Tailwind para estilos y Zustand para el estado global. Sin dependencias pesadas: React Router, Zustand y docx para exportar.

Estructura

src/
  main.jsx          # root de React
  App.jsx           # layout del doctor (sidebar + Outlet)
  router.jsx        # rutas lazy + guards de acceso
  store.js          # Zustand: auth, consulta, cola de visitas
  lib/
    api.js          # cliente fetch con JWT + manejo de 401
    docx.js         # genera el documento Word
    hooks/          # useRecorder, useTweaks, useVisitQueue…
  components/       # Header, Sidebar, WaveformCanvas, ui/, admin/
  pages/            # Login, Consent, Capture, Upload, Results, Patients…
    platform/       # sub-app del super admin

Cliente de API (lib/api.js)

Un único request() centraliza todas las llamadas. Inyecta el header Authorization con el token de localStorage, pone Content-Type: application/json salvo en FormData, y ante un 401 borra el token y dispara un evento cv:unauthorized que el store escucha para cerrar la sesión. Expone tres espacios de nombres: api (doctor), accountApi (owner) y adminApi (super admin).

async function request(path, opts) {
  const token = getToken();
  const res = await fetch(`${BASE}${path}`, {
    ...opts,
    headers: { ...(token ? { Authorization: `Bearer ${token}` } : {}), ...opts.headers },
  });
  if (res.status === 401) { setToken(null); window.dispatchEvent(new Event('cv:unauthorized')); }
  ...
}

Subida directa a Drive

uploadToDrive(sessionUri, blob) usa XMLHttpRequest (no fetch) para tener progreso real de subida: hace PUT del blob a la session_uri que devolvió el backend, reporta porcentaje vía onProgress y soporta cancelación con AbortSignal. El archivo nunca pasa por el backend.

Estado global (store.js)

Auth

user y hasToken hidratados desde localStorage. setAuth persiste; logout limpia todo y revoca blobs.

Consulta actual

consentAccepted, patient, consultType, blob/blobUrl, extraction. resetConsultation deja todo listo para la siguiente.

Cola de visitas

queue[] para rondas: addQueueItem, updateQueueItem, removeQueueItem, cada item con su estado y progreso.

Build (Vite). Servidor de desarrollo en el puerto 5191 (strictPort), alias @ → src, y code-splitting manual que separa react y zustand en chunks propios. Las rutas se cargan con lazy + Suspense para un bundle inicial ligero.
