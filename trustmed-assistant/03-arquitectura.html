Vista de sistema

Dos aplicaciones desplegadas por separado —un SPA de React y una API FastAPI— que se comunican por HTTP+JSON, sobre una base de datos Postgres y tres servicios externos de nube/IA.

Transcripción → datos estructurados.

Principio de diseño. El audio no atraviesa el backend: el navegador sube el blob directo a Drive con una sesión resumable que el backend solo inicia (ver Integraciones). El backend luego descarga ese archivo para procesarlo. El trabajo pesado —transcripción y extracción— corre como BackgroundTask de FastAPI tras responder al cliente, así que la petición de crear la consulta vuelve de inmediato y el frontend hace polling del estado.
