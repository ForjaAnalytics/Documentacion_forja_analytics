14 · Infraestructura

Operación

Cómo levantar el sistema en local, qué variables necesita y los puntos delicados que conviene conocer antes de operarlo.

Puesta en marcha local

# 1 · Base de datos (Postgres en :5433)
docker compose up -d        # dentro de Backend/

# 2 · Backend (FastAPI en :8000)
pip install -r requirements.txt
cp .env.example .env        # y completar las claves
alembic upgrade head        # aplica migraciones
uvicorn main:app --reload --port 8000

# 3 · Frontend (Vite en :5191)
npm install                 # dentro de Frontend/
npm run dev

Recuerda apuntar el frontend al backend con VITE_API_URL y agregar el origin del frontend (http://localhost:5191) a CORS_ORIGINS del backend.

Puntos delicados

ffmpeg debe estar instalado

Sin ffmpeg en el PATH el audio no se normaliza y Scribe puede alucinar sobre colas de silencio. En producción lo añade nixpacks.toml; en local instálalo aparte. El pipeline no se cae si falta (usa el audio original), pero la calidad baja.

La carpeta debe ser Shared Drive

GOOGLE_DRIVE_FOLDER_ID tiene que apuntar a una Shared Drive compartida con la service account como Content Manager. Si no, el boot registra que la SA no puede acceder y /api/uploads/init fallará con 503.

Hablantes invertidos

Se asume speaker_0 = MÉDICO. Si el audio empieza con el paciente, la extracción confunde nombres; se corrige con el mapeo speakers= del formateador.

Costo y modelo

El costo lo dominan los segundos de audio (Scribe) y los tokens de Claude. El prompt caching abarata schemas repetidos y claude-haiku-4-5 reduce ~3× el gasto frente a Sonnet con calidad similar en extracción.

Observabilidad. El backend loguea cada paso del pipeline (cuidarte.pipeline, cuidarte.extract, cuidarte.audio) y el uso de tokens por consulta. El panel de super admin agrega consultas, tokens y segundos de transcripción por account y periodo, útil para vigilar consumo y costos.
