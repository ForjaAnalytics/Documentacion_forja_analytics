05 · El sistema

Pipeline de IA

El proceso asíncrono (app/services/pipeline.py) que convierte un audio en Drive en una extracción estructurada. Corre como BackgroundTask y va actualizando status para que el frontend lo siga por polling.

1
Descarga
drive.download_file
trae el audio de Drive
2
Normaliza
audio.normalize_for_stt
ffmpeg: quita silencio
3
Transcribe
transcribe.transcribe
ElevenLabs Scribe
4
Diálogo
format_as_dialog
turnos por hablante
5
Extrae
extract.extract
Claude tool use
6
Guarda
status = ready
extraction + usage

Estados de la consulta

queued → transcribing → extracting → ready  ·  cualquier fallo → error

Cada transición se hace commit en la DB en el momento, de modo que el frontend ve el avance en tiempo casi real. Si algo lanza una excepción, el pipeline la captura, marca status = "error" y guarda el mensaje en la columna error.

El corazón, en código

def run(consultation_id: str) -> None:
    c = db.get(Consultation, consultation_id)

    # 1-2. Reutiliza transcripción cacheada si ya existe
    #       (útil al reextraer tras cambiar capture_config)
    if c.transcript and (c.transcript.get("words") or c.transcript.get("text")):
        transcript = c.transcript
    else:
        c.status = "transcribing"; db.commit()
        audio_bytes = drive.download_file(c.drive_file_id)
        audio_bytes, fname, mime = audio.normalize_for_stt(audio_bytes, ...)
        transcript = transcribe.transcribe(audio_bytes, filename=fname, mime_type=mime)
        c.transcript = transcript

    transcript_text = transcribe.format_as_dialog(transcript)

    # 3. Extracción con Claude usando el capture_config del usuario
    c.status = "extracting"; db.commit()
    user = db.get(User, c.user_id)
    extraction, usage = extract.extract(transcript_text, user.capture_config or {})
    c.extraction = extraction
    c.tokens_input = usage["input_tokens"] + usage["cache_read_tokens"] + usage["cache_write_tokens"]
    c.tokens_output = usage["output_tokens"]
    c.model_used = usage["model"]

    c.status = "ready"; c.ready_at = now(); db.commit()

Normalización de audio

Las grabaciones del navegador a veces traen una cola larga de silencio digital. Si se envía completa, Scribe alucina texto sobre el silencio y corrompe la transcripción. normalize_for_stt usa ffmpeg para quitar silencio (silenceremove a −45 dB) y reencoda a FLAC 16 kHz mono.

Fallback defensivo: si ffmpeg no está en el PATH o falla, devuelve el audio original sin tocar — el pipeline no se cae y el dev local sin ffmpeg sigue funcionando.

Transcripción cacheada

El paso 1–2 se salta si la consulta ya tiene un transcript guardado. Esto permite re-extraer (volver a llamar a Claude) tras cambiar el capture_config sin re-transcribir ni pagar de nuevo el audio — el script scripts/rerun_extract.py usa justamente esto.

Reconstrucción del diálogo

format_as_dialog agrupa las palabras de Scribe por speaker_id en turnos y antepone una etiqueta: MÉDICO: / PACIENTE:. Si no hubo diarización, cae al texto plano. Ese texto etiquetado es lo que recibe Claude.

Seguimiento de uso

Cada consulta guarda tokens_input, tokens_output, transcription_seconds y model_used. Estos contadores alimentan el reporte de uso del panel de super admin (ver Multitenant y roles).
