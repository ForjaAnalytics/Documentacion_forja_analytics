09 · Datos & API

API REST

FastAPI con siete routers bajo /api/*. Salvo el health-check, el login/registro y la lectura/aceptación de invitaciones, todo exige un JWT en el header Authorization: Bearer <token>.

Público / operacional

MétodoRutaDescripción
GET/healthLiveness — {"status":"ok","service":"cuidarte-voice-back"}. Usado por el healthcheck de Railway.

auth — registro, sesión, invitaciones

MétodoRutaDescripción
POST/api/auth/register-individualCrea account individual + owner. Provisiona carpeta de Drive. Devuelve JWT.
POST/api/auth/register-businessCrea account business + owner + subcarpeta del doctor.
POST/api/auth/registerCompat: alias de registro individual.
POST/api/auth/loginValida credenciales (bcrypt) y emite JWT. 403 si la cuenta está suspendida.
GET/api/auth/mePerfil del usuario autenticado.
PATCH/api/auth/meActualiza nombre y/o rota contraseña (current + new juntos).
GET/api/auth/invitations/{token}Público. Detalles de una invitación pendiente.
POST/api/auth/invitations/{token}/acceptPúblico. Acepta, crea el member y devuelve JWT.

uploads · consultations · me JWT

MétodoRutaDescripción
POST/api/uploads/initAbre una sesión resumable en Drive y devuelve session_uri + folder_id + filename.
POST/api/consultationsCrea la consulta (queued) con el drive_file_id y dispara el pipeline en background.
GET/api/consultations/{id}Estado + extracción de una consulta (objetivo del polling).
PATCH/api/consultations/{id}Guarda ediciones del médico sobre la extracción o el paciente.
GET/api/consultationsLista las consultas del doctor (filtro opcional por patient_id).
GET/api/me/capture-configEsquema de captura del usuario.
PUT/api/me/capture-configReemplaza el esquema de captura (validado con Pydantic).

patients JWT

MétodoRutaDescripción
GET/api/patientsLista con búsqueda por nombre/documento y conteo de consultas.
POST/api/patientsCrea paciente. 409 si repite documento del doctor.
GET/api/patients/{id}Ficha + conteo de consultas.
PATCH/api/patients/{id}Actualiza la ficha.
DELETE/api/patients/{id}Elimina el paciente y desvincula (no borra) sus consultas.
GET/api/patients/{id}/consultationsHistorial del paciente.

account — owner del account owner

MétodoRutaDescripción
GET / PATCH/api/accountDatos + uso del mes del account; editar nombre y capture_config_default.
GET/api/account/usersDoctores del account.
PATCH/api/account/users/{id}Cambia el status de un miembro (no al owner ni a uno mismo).
GET / POST/api/account/invitationsListar / crear invitaciones (solo business; respeta el cupo).
DELETE/api/account/invitations/{id}Revoca una invitación pendiente.
GET/api/account/overviewMetadata mínima por paciente/doctor (solo business).

admin — plataforma super_admin

MétodoRutaDescripción
GET / POST/api/admin/accountsListar (con filtros) / crear accounts.
GET / PATCH/api/admin/accounts/{id}Ver / editar cuotas y estado de un account.
GET / PATCH/api/admin/users{/id}Listar usuarios globales / editar uno.
GET/api/admin/usageUso agregado: consultas, tokens, segundos de transcripción.
GET/api/admin/billingResumen por estado de account (placeholders de Stripe).
POST/api/admin/impersonate/{user_id}Token corto (1 h) para actuar como otro doctor.
