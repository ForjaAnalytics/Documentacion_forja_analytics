Stack & deployment

Contenedores Docker sobre una imagen base de Playwright. Producción en Railway; frontend en Vercel; sin remote de GitHub.

Backend

Python 3.11+ · FastAPI · Uvicorn · httpx (async) · asyncpg / aiosqlite · Playwright · BeautifulSoup + markdownify (HTML→MD) · bcrypt + PyJWT.

Imagen base: mcr.microsoft.com/playwright/python:v1.47.0-jammy — los 5 servicios comparten layers.

Frontend

React 18 · TypeScript 5 · Vite 5 · TailwindCSS 3 · Recharts · framer-motion · lucide-react.

Build: tsc -b && vite build → Vercel.

Modelo de deploy. No hay GitHub remote. Se despliega con railway up --service <nombre>, que sube todo el working tree local (respetando .railwayignore). Los archivos sin commitear en local son los que corren en producción — el workflow es deployar sin commit hasta alcanzar un milestone. El .env nunca va al repo.
