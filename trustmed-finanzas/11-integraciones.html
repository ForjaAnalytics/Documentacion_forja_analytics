Integraciones externas

El sistema apoya en cinco servicios o librerías externas. Solo una llama a una API ajena: Ollama, y corre en local. El resto son librerías que el backend usa para generar archivos o el frontend para visualizar.

Ollama Generación de planes con IA local

Servidor de modelos LLM corriendo en la misma red. El backend lo consume por HTTP plano (http://localhost:11434 por defecto) con el SDK ollama (0.4.4). El modelo por defecto es Mistral, pero es configurable.

VariablePara quéDefault
OLLAMA_HOSTURL del servidor de Ollama.http://localhost:11434
OLLAMA_MODELModelo a usar.mistral
OLLAMA_TEMPERATURACreatividad de la respuesta (0–1).0.7
OLLAMA_TIMEOUTTiempo máximo de generación en segundos.120

Privacidad: los datos del consultorio (facturación, costos, márgenes) viajan al modelo dentro del prompt pero no salen a una API de terceros. Si Ollama no responde, la generación falla y se devuelve un 503; cada intento queda en GeneracionIALog con su error.

Leaflet Mapa competitivo

leaflet + react-leaflet + react-leaflet-cluster en el frontend, para el módulo TrustMap. Tiles públicos por defecto; el dataset de competidores se sirve desde el backend o desde un JSON estático en el MVP. No requiere API key.

ReportLab + Matplotlib PDFs financieros

El endpoint GET /reportes/pdf arma el informe con reportlab e incrusta gráficas generadas con matplotlib. El resultado se descarga directamente desde el navegador.

Pandas + Openpyxl Plantilla y parser de Excel

La carga masiva de facturas usa openpyxl para generar y leer la plantilla, y pandas para limpiar y validar las filas antes de persistirlas. La plantilla se descarga desde GET /facturas/template/download.

Email Pendiente

Existe un email_service.py esqueleto para envíos transaccionales (reset de contraseña, notificaciones, aprobación de planes). Aún no está integrado en los flujos; queda como gancho para cuando se conecte un proveedor (SES, SendGrid, etc.).

Resumen de claves y configuración

Todas las credenciales viven en variables de entorno (ver Operación). Si una falta, el endpoint que la necesita devuelve un error explícito en lugar de fallar en silencio.
