09 · Datos & API

API REST

FastAPI con prefijo /api/v1. Salvo el health-check, el login y el registro, todo exige un JWT en el header Authorization: Bearer <token>. Documentación interactiva en /api/docs (Swagger UI) y /api/redoc.

planes — roadmap jerárquico JWT

MétodoRutaDescripción
POST/planesCrea un plan: arma el prompt, llama a Ollama y persiste el JSON jerárquico.
GET/planesLista los planes del consultorio.
GET/planes/{id}Detalle completo con milestones, fases y tareas.
PUT/planes/{id}Editar plan.
PATCH/planes/{id}/progresoActualiza progreso (recalcula en cascada).
Sub-rutas anidadas · /planes/{id}/milestones, …/milestones/{mid}/fases, …/fases/{fid}/tareas con sus POST y PUT correspondientes.

Esquema y validación. Cada endpoint tiene su par de schemas Pydantic en app/schemas/ que documenta entrada y salida en Swagger automáticamente. Los errores se centralizan en middlewares/error_handler.py para no filtrar información sensible.
