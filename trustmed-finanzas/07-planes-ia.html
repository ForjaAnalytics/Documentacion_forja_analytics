07 · El sistema

Planes de acción con IA

El médico responde un cuestionario de contexto; Ollama genera un roadmap jerárquico —Plan → Milestones → Fases → Tareas— ajustado al consultorio. El sistema lo persiste, lo presenta editable y mantiene el progreso al día.

Generación con Ollama

El servicio plan_service arma un prompt a partir del PlanContexto, los datos operativos del consultorio y los KPIs actuales, y llama al servidor de Ollama:

POST http://localhost:11434/api/generate
{
  "model":       "mistral",
  "prompt":      "<contexto + plantilla del roadmap>",
  "temperature": 0.7,
  "format":      "json"
}

La respuesta es un JSON con la jerarquía completa (plan + milestones + fases + tareas). El servicio la valida con Pydantic y la persiste en una transacción. Si Ollama no responde o devuelve un JSON inválido, se aborta y se devuelve un 503 con el detalle.

Auditoría: GeneracionIALog

Cada llamada al modelo deja un registro con:

  • modelo_usado, temperatura
  • prompt_enviado y respuesta_cruda
  • datos_analisis (los KPIs que se mandaron como contexto)
  • tiempo_generacion_ms
  • exitoso, error_mensaje, advertencias

Sirve para depurar prompts, comparar versiones del modelo y rastrear por qué un plan salió como salió.

Progreso en cascada

El progreso siempre fluye de abajo hacia arriba:

Tareas → Fases → Milestones → Plan

Cuando una tarea pasa a completada, el sistema recalcula el porcentaje de su fase; con eso, el del milestone; y con eso, el del plan. Las dependencias entre tareas (PlanTareaDependencia) permiten bloquear una tarea hasta que se complete su predecesora.

KPIs por nivel

Cada nivel guarda su propio JSON de KPIs:

  • Plan — kpis_baseline (cómo estaba al empezar), kpis_target (a dónde quiere llegar), kpis_actual (estado de hoy).
  • Milestone — kpis específicos del hito.
  • Tarea — metricas_antes y metricas_despues para cuantificar el impacto de cada tarea ejecutada.

UI · páginas relacionadas

RutaFunción
/generar-planFormulario de contexto (FormularioPlanContexto) y disparo de la generación.
/planesLista de planes existentes con su estado y progreso.
/planes/:idDetalle del plan: navegación por milestones/fases/tareas, marcado de avance y métricas.

Por qué IA local. Ollama corre dentro de la infraestructura del usuario, así que los datos del consultorio (facturación, costos, márgenes) no salen a una API de terceros. El modelo, la temperatura y el timeout son configurables por entorno (OLLAMA_MODEL, OLLAMA_TEMPERATURA, OLLAMA_TIMEOUT).
