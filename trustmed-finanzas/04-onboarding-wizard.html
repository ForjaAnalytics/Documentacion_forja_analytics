04 · El sistema

Onboarding y wizard

El recorrido desde el registro hasta tener el consultorio configurado y listo para cargar facturas. Cinco pasos con valores por defecto inteligentes y persistencia incremental.

Registro

El usuario se crea con email + contraseña + nombre. POST /api/v1/auth/register crea el User y, según el flujo, también un Consultorio inicial. La contraseña se guarda solo como hash bcrypt; el login devuelve un JWT y los datos básicos de usuario y consultorio.

Paso 1 · Datos operativos

Horas semanales de atención, días laborables, capacidad máxima por jornada, tiempo medio por consulta, margen objetivo y la especialidad del consultorio. La especialidad es la pieza clave: dispara los defaults de los pasos siguientes.

Paso 2 · Costos mensuales

Nómina (médico, enfermera, auxiliares), costos fijos (arriendo, servicios, software) y costos variables. La Configuracion que se persiste sirve después a los reportes para calcular márgenes reales por procedimiento y por tipo de pagador.

Paso 3 · Procedimientos

Cada procedimiento lleva nombre, código, tiempo en minutos, costo de materiales y precio sugerido. El sistema arranca con un catálogo por especialidad (industy_defaults_service) que el usuario edita; los nuevos se crean al vuelo.

Paso 4 · Tarifas por pagador

Para cada combinación procedimiento × tipo de pagador (EPS, prepagada, particular…), el usuario define el precio y el copago. De ahí salen los márgenes que alimentan el módulo de pricing intelligence.

Defaults por especialidad

El endpoint GET /wizard/defaults?especialidad=… devuelve valores razonables para que el usuario no tenga que pensar el formulario en frío: procedimientos típicos, tarifas medias, costos de referencia. El usuario los ajusta a su realidad antes de confirmar.

# GET /api/v1/wizard/defaults?especialidad=cardiologia
{
  "procedimientos": [
    { "nombre": "Consulta de primera vez", "tiempo_minutos": 30, "precio_sugerido": 180000 },
    { "nombre": "Electrocardiograma", "tiempo_minutos": 15, "precio_sugerido": 95000 },
    ...
  ],
  "costos_referencia": { "medico_mensual": 8500000, "arriendo": 2200000, ... },
  "tarifas_promedio": { "eps": 0.65, "prepagada": 0.85, "particular": 1.0 }
}

Estado del wizard en el frontend

Un wizardStore (Zustand) guarda el paso actual y los datos parciales para que el usuario pueda navegar entre pasos sin perder lo ya capturado. Cada paso confirma contra el backend antes de avanzar, así que aunque se cierre el navegador, el progreso vive en la base de datos.

Tiers de suscripción

TierLímitesAcceso
freeHasta 30 facturas históricas, 1 consultorio.Wizard, facturas, caja básica y dashboard. Bloqueo en pricing intelligence y planes con IA.
premiumFacturas ilimitadas.Todo lo anterior + benchmark de precios, simulaciones, TrustMap completo y planes generados con IA.
enterpriseMulti-consultorio, soporte directo.Todo lo de premium con onboarding asistido y cuotas configurables.

El gatekeeping se hace en el backend (servicio bloquea o devuelve 403 si excede el límite) y en el frontend con PremiumOverlay, que muestra una pantalla translúcida con la invitación a actualizar.
