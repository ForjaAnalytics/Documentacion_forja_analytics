Frontend

SPA en React 19 + TypeScript, sobre Vite, con Zustand para estado global, Tailwind para estilos y i18next para multi-idioma. Mapas con Leaflet, gráficas con ApexCharts y Recharts.

Cliente de API (services/api.ts)

Axios con un interceptor que inyecta el header Authorization: Bearer con el JWT y otro que, ante un 401, limpia el store y redirige a /login. La URL base se resuelve por VITE_USE_LOCALHOST + VITE_PRODUCTION_API_URL/VITE_LOCALHOST_API_URL, así dev y prod conviven sin tocar código.

Estado global (Zustand)

authStore

user, consultorio, token, isAuthenticated, isLoading, isPremium. Persiste el token y el perfil para que la sesión sobreviva al refresh.

wizardStore

Paso actual, datos parciales por paso y banderas de validación. El backend sigue siendo la fuente de verdad: cada paso confirma antes de avanzar.

languageStore

Idioma actual (es / en). El LanguageToggle dispara el cambio y i18next recarga los textos sin recargar la SPA.

ProtectedRoute aplica el guard de autenticación; PremiumOverlay bloquea visualmente las funciones premium en el tier free con una invitación a actualizar.

Componentes y herramientas notables

Mapas — components/map/

MapView con react-leaflet + clustering; DoctorCard, FilterPanel, RadiusSelector, StatisticsPanel, RecommendationsPanel.

Gráficas

ApexCharts y Recharts para KPIs, evolución temporal y desglose por categoría.

Formularios

React Hook Form + Zod para validación tipada en el cliente. CurrencyInput formatea pesos colombianos al vuelo.

Guía interactiva — components/guia/

9+ piezas: hero, quick start, FAQ, glosario, búsqueda, atajos de teclado y progress tracker. Datos en data/guiaData.ts.

Build (Vite 7). npm run dev levanta el dev server; npm run build hace tsc + vite build. La configuración de Tailwind vive en tailwind.config.js y el lint en eslint.config.js.
