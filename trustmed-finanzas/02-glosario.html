Glosario

Doce términos que aparecen una y otra vez en el resto del documento. Si algo no te cuadra más adelante, probablemente esté definido aquí.

Consultorio Tenant

La unidad de aislamiento del sistema. Un User es dueño de uno o varios consultorios; todos los datos —facturas, caja, procedimientos, planes— cuelgan de un consultorio. Tiene un plan de suscripción (plan_suscripcion) que define los límites del tier.

Tier Plan de suscripción

free · premium · enterprise. El plan limita cuántas facturas se pueden cargar (p. ej. 30 en free) y el acceso a funciones avanzadas (IA, reportes premium). El frontend muestra un PremiumOverlay cuando se topa con el límite.

Wizard Onboarding en 5 pasos

El flujo de configuración inicial: (1) datos operativos, (2) costos mensuales, (3) procedimientos, (4) tarifas por pagador, (5) confirmación. Cada paso usa valores por defecto según la especialidad.

Tipo de pagador Quién paga

EPS, prepagada, particular, ARL, etc. Tiene un porcentaje_reembolso esperado y un tiempo_promedio_pago_dias. Sobre él se montan las tarifas de cada procedimiento.

Procedimiento + tarifa Servicio y su precio

Procedimiento describe el servicio (nombre, código, tiempo, costo de materiales, precio sugerido). ProcedimientoTarifa guarda el precio real por cada combinación procedimiento × tipo de pagador, con su copago.

Factura Lo que se debe cobrar

Una transacción facturable. Guarda valor_facturado (lo emitido) frente a valor_pagado (lo cobrado realmente) y avanza por estados: pendiente → pagada · anulada · vencida. La carga masiva exige mínimos para que el análisis sea representativo.

Caja Flujo real de dinero

Módulo separado de facturas. Maneja cuentas (efectivo, banco, tarjeta), movimientos (entradas/salidas con medio de pago y categoría), transferencias entre cuentas y conciliación con el banco. El saldo se recalcula a cada movimiento.

Pricing intelligence Inteligencia de precios

Compara los precios del consultorio contra los del mercado en un radio configurable (5–100 km) y simula el impacto de cambios de precio en volumen e ingresos, en tres escenarios: conservador, moderado, optimista.

TrustMap Mapa competitivo

Vista geográfica de la competencia con Leaflet: cada doctor o clínica aparece como un punto, filtrable por especialidad, rango de precios y radio. Acompañado de panel de estadísticas y recomendaciones por zona.

Plan de acción Roadmap del consultorio

Lo que el sistema sugiere hacer. Estructura jerárquica: PlanAccion → PlanMilestone → PlanFase → PlanTarea (con subtareas y dependencias). Tiene KPIs baseline / target / actual y un progreso que cascadea de las tareas hacia arriba.

Ollama IA local

Servidor local de LLM (modelo por defecto: mistral) que el backend usa para generar los planes. Cada generación se registra en GeneracionIALog con prompt, respuesta cruda, tiempo y errores para auditoría.

Plan contexto Entrada del plan

El cuestionario previo a generar un plan: región, si el consultorio tiene base de datos de clientes, si hace marketing, si tiene CRM, si maneja descuentos o promociones, si vende productos secundarios, etc. Es el contexto que recibe Ollama.

Aclaración de nombres. El producto se llama Trustmed Finanzas, pero el repositorio del backend usa internamente FinanzasBackEnd. Ambos nombran el mismo sistema. Convive con Trustmed Assistant dentro del ecosistema Trustmed, pero son aplicaciones independientes (no comparten autenticación ni base de datos).
